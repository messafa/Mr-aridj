Merge Sort Algorithm

Divide & Conquer
Sorting

Objective: Implement the Merge Sort algorithm demonstrating the Divide and Conquer paradigm with guaranteed O(n log n) performance.

private static void merge(int[] array, int[] left, int[] right) {
  int i = 0, j = 0, k = 0;
  // Merge two sorted halves
  while (i < leftSize && j < rightSize) {
    if (left[i] <= right[j]) {
      array[k++] = left[i++];
    } else {
      array[k++] = right[j++];
    }
  }
}

Algorithm Analysis

Time Complexity: O(n log n) in worst, average, and best cases

Space Complexity: O(n) auxiliary space

Stability: Stable (preserves relative order of equal elements)

Key Insight: The merge step is where two pre-sorted arrays are combined into a single sorted array, representing the "conquer" phase of the algorithm.

Backtracking in 2D Matrix

Backtracking
Pathfinding

Objective: Implement a recursive backtracking algorithm to find a path through a binary matrix from start to end point.

private static void traverseMatrix(int[][] matrix, int[][] visited, Point start, Point end) {
  // Base case: reached destination
  if (start.x == end.x && start.y == end.y) {
    System.out.println("Reached end point");
    return;
  }
  
  // Mark current cell as visited
  visited[start.y][start.x] = 1;
  
  // Explore four directions (DFS)
  traverseMatrix(matrix, visited, new Point(x, y-1), end); // Up
  // ... other directions
  
  // Backtrack: unmark for other paths
  visited[start.y][start.x] = 0;
}

Algorithm Analysis

Search Type: Depth-First Search with backtracking

Time Complexity: O(4^(m√ón)) in worst case (exponential)

Space Complexity: O(m√ón) for visited matrix + recursion stack

Critical Mechanism: The visited matrix prevents infinite loops, while the backtracking step (unmarking) allows exploration of alternative paths when dead ends are encountered.

Sudoku Solver

Constraint Satisfaction

Objective: Solve a standard 9√ó9 Sudoku puzzle using recursive backtracking with constraint checking.

public static boolean solve(int[][] board) {
  for (int row = 0; row < 9; row++) {
    for (int col = 0; col < 9; col++) {
      if (board[row][col] == 0) {
        // Try numbers 1-9
        for (int num = 1; num <= 9; num++) {
          if (isValid(board, row, col, num)) {
            board[row][col] = num; // Try
            if (solve(board)) return true; // Recurse
            board[row][col] = 0; // Backtrack
          }
        }
        return false; // No valid number
      }
    }
  }
  return true; // Solved
}

Algorithm Analysis

Problem Type: Exact Cover Problem with constraints

Worst Case Time: O(9^(n)) where n is number of empty cells

Constraint Checking: Three simultaneous constraints: row, column, and 3√ó3 box

Optimization: The isValid() function prunes invalid branches early, significantly reducing search space

Alpha-Beta Pruning

Game Theory

Objective: Enhance Minimax with Alpha-Beta pruning to reduce the number of nodes evaluated without affecting optimality.

static int alphaBeta(Node node, int depth, int alpha, int beta, boolean maximizingPlayer) {
  if (maximizingPlayer) {
    int maxEval = Integer.MIN_VALUE;
    for (Node child : node.children) {
      int eval = alphaBeta(child, depth-1, alpha, beta, false);
      maxEval = Math.max(maxEval, eval);
      alpha = Math.max(alpha, eval);
      // ‚úÇÔ∏è PRUNING CONDITION
      if (beta <= alpha) break;
    }
    return maxEval;
  }
  // Minimizing player similar with beta update
}

Algorithm Analysis

Optimization: Can reduce evaluated nodes by 50% or more in game trees

Alpha (Œ±): Best already explored option for maximizer

Beta (Œ≤): Best already explored option for minimizer

Pruning Rule: When Œ≤ ‚â§ Œ±, the branch cannot affect final decision

Efficiency: Same result as Minimax but much faster

Branch and Bound

Optimization
Combinatorial

Objective: Solve the 0/1 Knapsack problem using Branch and Bound with pruning based on upper bound calculations.

static void knapsack(int index, int weight, int value, int capacity, Item[] items) {
  if (weight > capacity) return;
  maxValue = Math.max(maxValue, value);
  if (index == items.length) return;
  
  // Calculate upper bound (optimistic estimate)
  int b = bound(index, weight, value, capacity, items);
  
  // ‚úÇÔ∏è PRUNE if bound ‚â§ current best
  if (b <= maxValue) return;
  
  // Branch: take item
  knapsack(index+1, weight+items[index].weight,
            value+items[index].value, capacity, items);
  // Branch: skip item
  knapsack(index+1, weight, value, capacity, items);
}

Algorithm Analysis

Problem: NP-Hard combinatorial optimization

Strategy: Depth-first search with intelligent pruning

Bound Function: Relaxes integrality constraint for optimistic estimate

Pruning Power: Eliminates branches that cannot beat current best solution

Real-world Use: Resource allocation, portfolio optimization, cutting stock

Functional Programming in Lisp

Functional
Recursion

Objective: Write a recursive Lisp function to sum all numbers in a nested list structure.

(defun sum-all (lst)
  (cond
    ((null lst) 0) ; Base case
    ((listp (car lst)) ; If first element is list
     (+ (sum-all (car lst)) ; Recurse on sublist
       (sum-all (cdr lst)))) ; Recurse on rest
    ((numberp (car lst)) ; If first is number
     (+ (car lst) ; Add number
       (sum-all (cdr lst)))) ; Recurse on rest
    (t (sum-all (cdr lst))))) ; Ignore non-numbers

Algorithm Analysis

Paradigm: Functional programming with pure functions

List Processing: Uses car (first) and cdr (rest) operations

Recursion: Natural approach for nested structures in Lisp

Pattern Matching: cond statement handles different data types

Educational Value: Demonstrates thinking recursively about data
